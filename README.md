## 반응형 규칙

- 콘텐츠는 `max-width: 1400px`로 중앙 정렬되며, 그보다 넓은 화면에서는 여백이 늘어나고 좁은 화면에서는 뷰포트 폭에 맞춰 반응형으로 줄어듭니다.
- 모든 페이지에서 가로 스크롤이 발생하지 않도록 `overflow-x: hidden`을 (모바일 대응 이후) `html` 요소에만 적용했습니다 (`body`에 걸면 이중 스크롤바가 생기는 브라우저 이슈가 있어 분리함).
- 900px 이하에서는 원래 좌우 여백이 없던 섹션들에 공통으로 20px 패딩을 추가했습니다.
- 720px 이하에서는 상단 네비게이션이 햄버거 메뉴로 전환되고, S0 히어로 텍스트가 중앙 정렬 + 버튼이 하단 고정되도록 레이아웃이 바뀝니다.

## 인터랙션

- **스크롤 fade-in**: 각 섹션의 콘텐츠 wrapper에 `.reveal` 클래스를 부여하고 `IntersectionObserver`로 뷰포트 진입을 감지해 `in-view` 클래스를 추가합니다 (`opacity: 0 → 1`, `translateY → 0`, 한 번만 재생). 요소별로 `data-reveal-threshold`를 지정하면 더 늦게(화면에 더 많이 들어왔을 때) 트리거되도록 개별 조정할 수 있습니다. `prefers-reduced-motion: reduce` 환경에서는 애니메이션 없이 즉시 표시됩니다.
- **S0 히어로**: 배경 영상 로드가 끝나면(`readyState`/`loadeddata`) 로고·네비·문구·버튼이 순서대로 페이드업됩니다. PC 네비게이션은 스크롤해도 상단에 고정되어 있습니다.
- **S2 파이차트**: `stroke-dasharray` 기반 SVG 도넛 차트로, 슬라이스 또는 범례에 마우스를 올리면 해당 조각이 강조되고 중앙 텍스트가 실시간으로 전환됩니다. 마우스 좌표 → 중심 기준 각도 계산으로 hover 대상을 판별합니다(SVG dasharray는 마우스 히트테스트를 반영하지 않는 브라우저 특성 때문).
- **S3 성장 그래프(PC)**: 스크롤 진입 시 (1) SVG 곡선이 `getTotalLength()` 기반으로 왼쪽에서 오른쪽으로 그려지고 (2) 로봇 1.0→2.0→3.0 이미지가 순서대로 나타나고 (3) 34/88/330 수치가 0에서부터 카운트업되고 (4) 배경의 연한 화살표가 아래에서 위로 자라나는 4단계 시퀀스로 재생됩니다.
- **S7/S9 캐러셀**: 마우스 드래그와 휠 스크롤로 이동 가능하며, S9는 카드 세트를 한 번 복제해 `requestAnimationFrame`으로 스크롤 위치를 계속 이동시키다가 절반 지점에서 앞으로 되감아 무한 롤링처럼 보이게 했습니다 (마우스 올리거나 드래그 중일 땐 정지).
- **S8 하이브리드 매장**: 이전/다음 버튼을 누르면 현재 이미지는 반대 방향으로 슬라이드 아웃되고 새 이미지가 반대편에서 슬라이드 인되는 페이지 전환 효과가 적용됩니다.
- **플로팅 버튼**: 우측 하단 상담신청/TOP 버튼은 스크롤이 S2 섹션에 도달하는 순간부터 페이드인되어 계속 노출됩니다.

## 실행 방법

`b-eat-landing/index.html` 파일을 브라우저에서 직접 열면 대부분의 화면은 확인할 수 있지만, **유튜브 임베드(S6-1)와 배경 영상(S0)은 `file://` 경로에서는 정상 동작하지 않을 수 있으므로** 아래 로컬 서버로 여는 것을 권장합니다.

저장소 루트의 `serve.ps1`을 PowerShell로 실행하면 `b-eat-landing/`을 정적 파일 서버로 띄웁니다 (기본 포트 8737).

```powershell
./serve.ps1
```

이후 브라우저에서 `http://localhost:8737/index.html`로 접속하면 됩니다.
